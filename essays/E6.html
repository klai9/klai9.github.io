How was developing this assignment different than assignment #2?

This was different than assignment 2 because we were now using sessions to pass data, and although we had to implement less things that the previous two assignments, I found myself struggling a lot. It felt as if it took so much more time to write so little code.

When you ran into a problem, what did you do to address it?

If I ran into a problem, I would first look at my code to see if it's a spelling error, or something easy that I just went a little too fast on and didn't catch. I found that a lot of the times when I went to check my code, it gave me time to just slow down and carefully read through my code and I fixed a lot of my errors right then and there. If I still ran into trouble, I would go to my best friend, Google. I did a huge, huge amount of reading for assignment 3.

Describe what worked well in doing this assignment?

I found that the code examples we were given helped me a lot and worked well. The examples gave me structure to build upon because I did not have to start entirely from scratch. I feel like a lot of things went well, but the things that went wrong were extra bad in weight.

Describe what did not work well in doing this assignment?

My individual requirement did not work well and my organization just went out the door. I gave myself a day to review and comment out all of the code, and that helped. There were many things that had the smallest fixes, but I spent hours or days mulling and thinking about the solution.

What did you learn from doing this assignment?

I learned that having a plan, being organized, and starting early is the optimal way of doing assignments. I learned how websites are made, and it has made me weirdly hyperaware when I visit websites now, I think about how each one works and how specific elements were implemented.

If you could go back in time and do things differently, what would you change?

I would have reached to Professor Kazman. I need to realize when something's not working and when to ask for help instead of failing after trying again and again to fix it myself. I also shouldn't have waited the day before this assignment was due to test it on the server because I have a small sudo npm problem that only happens in the itm server and I could've fixed it if I'd tested it sooner.
